Campus Second-hand Trading Platform – Task 1

📌 Project Overview

This is the core module (Task 1) of a campus second-hand trading platform, implemented in Python following Object-Oriented Programming principles. It demonstrates the use of Abstract Data Types (ADT) , encapsulation, inheritance, polymorphism, and modular design to manage users, books, and orders. The code provides a solid foundation for future extensions (GUI, database, etc.).

✨ Key Features

  • Abstract Base Class (ADT) : BaseUser defines a common interface (get_info, change_email) for all user types.
  • Inheritance & Polymorphism: Seller and Buyer inherit from User and override get_info() to include their specific attributes.
  • Encapsulation: Internal attributes (e.g., _name, _email) are accessed via methods, with validation (campus email suffix, non-negative prices).
  • Modular Data Management: DataHandler and its subclasses (UserHandler, BookHandler, OrderHandler) separate concerns for different data types.
  • State Management: Book supports condition updates (new, like_new, used, sold); Order supports status updates (pending, paid, shipped, completed, cancelled).
  • Robustness: Duplicate IDs are rejected with exceptions; negative prices raise ValueError.

🧱 Class Structure

1. User Classes

Class Description Key Methods
BaseUser (ABC) Abstract base for all users get_info(), change_email()
User Basic user implementation Implements above; __str__
Seller Seller with store name Overrides get_info()
Buyer Buyer with shipping address Overrides get_info()

2. Product & Order Classes

Class Description Key Methods
Book Represents a second-hand book update_condition(), get_book_details()
Order Represents a transaction update_status(), get_order_details()

3. Data Managers

Class Description
DataHandler Generic manager for any item type (add/remove/get_all)
UserHandler Specialized for User objects
BookHandler Specialized for Book objects
OrderHandler Specialized for Order objects
